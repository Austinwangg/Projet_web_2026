Bugs corrigés — Sprint du 28 mai 2026
⚡ Bug 1 — Mauvaises données dans Mes Réservations
Symptôme : Après réservation d'une activité seule (sans transport/hébergement), "Mes Réservations" affichait la mauvaise destination (ex: Shanghai par défaut).

Cause : L'item activité ajouté au panier ne portait pas destSlug. ScreenPayment se rabattait alors sur detailId (dernière page de détail visitée) pour résoudre la destination_id.

→ Fix : Ajout de destSlug: cartModal.slug et dateDepart: cartDate dans l'item panier de ScreenActivites.jsx.
⚡ Bug 2 — Calendrier Transport ne suit pas la souris
Symptôme : Dans la page Transport, le calendrier de sélection de dates s'affichait mal (position incorrecte) et ne recalculait pas sa position lors du déplacement de la souris.

Cause : Le popoverStyle était initialisé à {} (position: absolute CSS), puis mis à jour en position: fixed dans un useEffect → flash et décalage. Le clamp horizontal manquait aussi (débordement du bord droit).

→ Fix : Calcul de la position avant ouverture (dans le onClick) via calcPopoverStyle(), clamp bord droit, et maxWidth: 95vw dans DateRangePicker.jsx.

Pistes d'amélioration prioritaires
PRIORITÉ HAUTE
Annulation individuelle du transport
Ajouter un bouton "Retirer le transport" par entrée dans "Mes Voyages". Backend : PUT /reservations?id=X avec {"transport_id": null} + restitution des places.
PRIORITÉ HAUTE
Voyageurs nominatifs → BDD
La table reservation_voyageurs existe. Il suffit d'ajouter un endpoint POST/DELETE pour persister les noms saisis dans ScreenItinerary.jsx.
PRIORITÉ MOYENNE
Favoris persistés en BDD
Créer une table favoris (utilisateur_id, destination_id). Synchronisation au login. Actuellement perdu si on change de navigateur.
PRIORITÉ MOYENNE
Génération PDF d'itinéraire
Bouton "Télécharger PDF" dans ScreenItinerary.jsx via window.print() avec CSS @media print dédié. Très visible lors de la démo.
PRIORITÉ MOYENNE
Météo sur les pages destination
Widget météo dans ScreenDetail.jsx via API OpenWeatherMap (gratuite). Affiche la météo de la ville cible. +1 point bonus.
PRIORITÉ BASSE
Pagination des résultats
Actuellement tous les résultats sont chargés d'un coup. Ajouter une pagination côté backend (LIMIT/OFFSET) améliorerait les performances et l'UX.
PRIORITÉ BASSE
Comparaison de séjours
Permettre de sélectionner 2–3 destinations et afficher un tableau comparatif (prix, durée, avis, activités). Fonctionnalité bonus +1 pt.
PRIORITÉ BASSE
Destination manquante sur activité seule
Si activite.slug est null en BDD, le paiement peut échouer pour une activité sans transport ni hébergement. S'assurer que chaque destination a un slug non vide.
