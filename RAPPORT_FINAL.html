POINTS D'ATTENTION

Ce qui reste à améliorer

⚠ Annulation trajet individuel — 1 pt à risque
L'annulation d'un trajet réservé fonctionne via l'annulation complète de la réservation, mais il n'y a pas de bouton dédié "annuler uniquement le transport" dans ScreenAccount.

Fichier : ScreenAccount.jsx
Action : Ajouter un bouton d'annulation de transport séparé sur les réservations confirmées.
⚠ Favoris semi-statiques — bonus partiel
L'onglet Favoris dans Mon Compte affiche 4 destinations codées en dur depuis data.js. Les favoris ne sont pas sauvegardés en base ni liés à l'utilisateur connecté.

Fichier : ScreenAccount.jsx
Action : Brancher sur une table favoris ou accepter le 0 pt bonus sur ce point.
ℹ Navigation — trop de liens dans la nav
Le Header affiche maintenant 6 liens (Destinations, Séjours, Hébergements, Activités, Transport, Itinéraire). Sur petits écrans cela peut déborder.

Fichier : Header.jsx
Action : Vérifier sur la résolution de la salle de démo.
ℹ ScreenActivites — slug manquant
La page Activités affiche un bouton "Voir le séjour →" mais le champ slug n'est pas retourné par l'API /activites, donc le bouton ne s'affiche pas toujours.

Fichier : ScreenActivites.jsx + Activite.php
Action : Joindre la table destinations dans la requête SQL.
✕ Image réelle des destinations — galerie vide
Les images de destinations utilisent un composant Placeholder (couleur de fond). Aucune vraie photo n'est chargée. Pour la démo, les placeholders fonctionnent mais c'est visuellement moins convaincant.

Action : Optionnel — ajouter des vraies URLs d'images dans la colonne image_url de la table destinations.
✓ Port MAMP configuré (81)
Le proxy Vite est désormais configuré sur le port 81 dans vite.config.js. À vérifier si le projet est déplacé sur une autre machine — le port peut différer.

Fichier : frontend/vite.config.js
PRÉPARATION

Checklist démo

À vérifier le jour de la démonstration, dans l'ordre de passage devant le jury.

  • ✓
    MAMP lancé — Apache port 81, MySQL port 3306 actifs. Base voyagevista avec données.
  • ✓
    Frontend démarré — npm run dev dans frontend/. Ouvrir localhost:3000.
  • !
    Compte admin prêt — avoir un compte avec rôle admin en base pour démontrer ScreenAdmin (ajout/suppression gestionnaire).
  • !
    Compte user prêt — avoir un compte avec au moins une réservation confirmée pour montrer modification, annulation activité, notifications.
  • ✓
    Navigation générale — cliquer sur tous les liens du header : Destinations, Hébergements, Activités, Transport, Itinéraire.
  • ✓
    Flux réservation complet — Accueil → Destination → Séjour → Panier (modifier un élément) → Paiement → Confirmation → Notification reçue.
  • ✓
    Modification réservation — Dans Mon Compte → Réservations → bouton ✎ Modifier → changer dates + voyageurs → sauvegarder.
  • ✓
    Annulation activité individuelle — Dans Mon Compte → Réservations → bouton "✕ retirer" sur une activité sans annuler tout le séjour.
  • ✓
    Voyageurs nominatifs — Aller dans Itinéraire → "Gérer les voyageurs" → ajouter prénom/nom → affichage dans la liste.
  • ✓
    Dark mode + bilinguisme — Toggle ◑ pour le thème, bouton FR/EN pour la langue. Toutes les pages doivent changer.
  • ✓
    Admin — ajout/suppression offre — Se connecter en admin → ScreenAdmin → Wizard 4 étapes pour créer une destination → bouton Supprimer.
  • !
    Disponibilités — Montrer qu'un hébergement complet est bloqué (badge COMPLET) et qu'une activité sans places est désactivée.
