Priorités recommandées (points les plus faciles à récupérer)

  1. Modification du profil — Ajouter un endpoint PUT /api/users/{id} et le brancher sur l'UI existante (2 pts)
  2. Notifications — Créer la table DB + 3 endpoints (create, list, mark-as-read) + déclencher après chaque réservation (4 pts)
  3. Annulations (trajet / hébergement / activité) — L'API réservations a déjà un PUT pour changer le statut, il faut brancher les boutons UI (6 pts)
  4. Réservation hébergement / inscription activité — Brancher la sélection UI sur les endpoints existants (4 pts)
  5. Sauvegarde des itinéraires — Créer une table itineraires + endpoints CRUD (10 pts)
  6. Admin : gestion des rôles — Utiliser le champ role déjà en DB, ajouter l'UI dans le panel admin (3 pts)
