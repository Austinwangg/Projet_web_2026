01

Activités — Dates et nombre de personnes à l'ajout panier

CRITIQUE · 2 pts

Actuellement le bouton "+ Panier" sur la page Activités ajoute l'activité sans demander de date ni de nombre de personnes. Or la grille d'évaluation demande explicitement la gestion des capacités (vérification que le nombre de places demandées ≤ places restantes) et l'inscription à une activité (qui implique une date et un nombre de participants).

À faire : Ouvrir une mini-modale au clic sur "+ Panier" avec un sélecteur de date + nombre de personnes. Vérifier côté backend que places_restantes ≥ nb_personnes avant de valider. Décrémenter les places après confirmation.

Comportement attendu (référence Airbnb Experiences, GetYourGuide) : sélecteur de date du calendrier, compteur de participants, prix recalculé en temps réel (prix × nb), bouton "Confirmer et ajouter".

AMÉLIORATION · +1 bonus

03

Itinéraire — Refonte UX : navigation par page dédiée

CRITIQUE · 2 pts

La solution actuelle avec modales inline dans la page Itinéraire est illisible et peu ergonomique : les listes sont compressées, les images absentes, les filtres inaccessibles.

Comportement attendu (référence TripAdvisor Trips, Roadtrippers) : quand l'utilisateur clique sur un bouton d'ajout (Séjour, Hébergement, Activité, Transport), le site navigue vers la page correspondante dans un mode "sélection". Lorsqu'il clique sur un élément, il est renvoyé automatiquement vers l'Itinéraire avec l'élément pré-ajouté.

À faire : Passer un flag fromItinerary: true dans les props de navigation. Chaque écran (Destinations, Hébergements, Activités, Transport) détecte ce flag et affiche un bandeau "Mode ajout à l'itinéraire". Le clic sur "Choisir" / "Ajouter" ajoute l'item dans un state partagé (ou localStorage) et redirige vers navigate('itinerary').

Détail technique itinéraire

La logique de navigation avec retour automatique s'implémente sans Redux ni state global complexe — juste un argument supplémentaire dans navigate().

Principe

Dans App.jsx, la fonction navigate(screen, args) accepte déjà un objet args. On y ajoute args.fromItinerary = true et args.pendingItem (l'item à ajouter au retour). Chaque écran de sélection reçoit une prop itineraryMode et un callback onAddToItinerary(item).

Changements fichier par fichier

FichierModification
App.jsx Ajouter state itineraryPending. Dans navigate(), si args.fromItinerary, sauver le callback. Passer itineraryMode + onAddToItinerary aux 4 écrans concernés.
ScreenItinerary.jsx Les 4 boutons font navigate('hebergement', { fromItinerary: true }) au lieu d'ouvrir une modale. Supprimer les modales inline.
ScreenHebergement.jsx Si itineraryMode, afficher bandeau jaune "Mode ajout à l'itinéraire". Remplacer bouton "Réserver" par "Choisir pour l'itinéraire" → appel onAddToItinerary(item) → navigate('itinerary').
ScreenActivites.jsx Idem. Si itineraryMode, afficher bandeau + remplacer "+ Panier" par "Choisir pour l'itinéraire".
ScreenTransport.jsx Idem. Si itineraryMode, le bouton "Sélectionner" → ajoute à l'itinéraire et redirige.
ScreenResults.jsx Idem pour les séjours. Si itineraryMode, le bouton "Voir le séjour" → "Ajouter à l'itinéraire".
